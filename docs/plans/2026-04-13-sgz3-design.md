# 三国志3网页版 - 项目设计方案

**创建日期：** 2026-04-13  
**项目名称：** SGZ3-Web  
**版本：** 1.0

---

## 一、项目概述

### 1.1 项目定位
完整复刻三国志3的核心玩法和功能，采用简化的像素风格素材，打造一款纯前端的单人策略游戏。

### 1.2 核心目标
- 忠实还原三国志3的游戏机制（内政、战争、外交、武将系统）
- 提供流畅的网页游戏体验
- 降低素材制作成本，采用像素风格
- 支持离线游玩，数据本地存储

### 1.3 技术选型
- **前端框架：** React 18 + TypeScript
- **UI方案：** Tailwind CSS + shadcn/ui
- **地图渲染：** Canvas 2D
- **状态管理：** Zustand
- **数据存储：** IndexedDB（存档） + JSON静态数据
- **AI系统：** 分阶段决策AI

## 二、核心功能模块

### 2.1 地图系统（优先开发）
- 2D俯视地图，Canvas渲染
- 显示三国时期的地理版图
- 城市据点标注（约40-50个城池）
- 势力范围用不同颜色区分
- 地形类型：平原、山地、水域、关隘
- 支持缩放和平移操作
- 天气显示：晴天、雨天、雪天

### 2.2 内政系统
- **城市管理：** 开发（增加人口）、商业（增加金钱）、防御（提升城防）
- **资源系统：** 金钱、粮草、兵员、民心
- **建设系统：** 市场、农场、兵营、城墙等设施
- **季节系统：** 春、夏、秋、冬影响资源产出
- **灾害系统：** 旱灾、水灾、瘟疫、蝗灾随机发生

### 2.3 武将系统
- **武将属性：** 统率、武力、智力、政治、魅力（1-100）
- **技能系统：** 每个武将有独特技能（如神算、勇将、仁德等）
- **忠诚度系统：** 影响武将是否叛变（0-100）
- **武将培养：** 通过战斗和内政提升能力
- **官职系统：** 封官提升武将能力和忠诚度
- **继承系统：** 君主死亡后继承机制

### 2.4 宝物系统
- **宝物类型：**
  - 武器类：青龙偃月刀、丈八蛇矛、方天画戟等（提升武力）
  - 书籍类：孙子兵法、太平要术等（提升智力）
  - 马匹类：赤兔马、的卢马等（提升移动或战斗优势）
  - 特殊物品：玉玺、和氏璧等（特殊效果）

- **获取方式：**
  - 城市搜索：派武将搜索城市可能发现宝物
  - 战斗缴获：击败敌将可能获得其宝物
  - 历史事件：特定事件触发获得

- **宝物效果：**
  - 属性加成：提升武将某项属性
  - 特殊技能：赋予武将特殊能力
  - 忠诚度影响：某些宝物影响武将忠诚

### 2.5 战争系统

#### 2.5.1 军队编制
- 兵种：步兵、骑兵、弓兵、水军
- 武将配置：每支军队可配置主将和副将
- 兵力上限：根据武将统率决定

#### 2.5.2 战斗类型
- **野战：** 开阔地带的遭遇战
- **攻城战：** 攻打敌方城市
- **水战：** 水域战斗，水军有优势

#### 2.5.3 战斗阵型系统
- **鱼鳞阵：** 防御型，防御+30%，攻击-10%
- **锋矢阵：** 攻击型，攻击+30%，防御-10%
- **雁行阵：** 平衡型，攻防平衡
- **方圆阵：** 包围型，对侧翼敌人攻击+20%
- **长蛇阵：** 机动型，移动速度+50%
- **衡轭阵：** 攻防兼备，攻防各+15%

#### 2.5.4 战术系统
- **埋伏：** 在特定地形隐藏，突袭敌人
- **火攻：** 对敌人造成持续伤害
- **水攻：** 在水域战斗时攻击+50%
- **奇袭：** 无视防御直接攻击

#### 2.5.5 计谋系统
- **离间：** 降低敌方武将忠诚度
- **伪报：** 使敌人撤退
- **混乱：** 使敌人混乱，降低攻击和防御
- **劝降：** 劝说敌方武将投降

#### 2.5.6 武将战斗技能

**顶级技能（传说级）：**
- **诸葛亮 - 神算：** 所有计谋成功率+50%，计谋伤害+30%，可连续使用计谋
- **吕布 - 勇将：** 攻击力+40%，单挑必胜，战斗时士气+20
- **关羽 - 武圣：** 攻击力+30%，对敌将攻击+50%，单挑时攻击+100%
- **张飞 - 猛将：** 攻击力+35%，对士气低落敌人攻击+100%
- **周瑜 - 火攻：** 火攻伤害+100%，火攻范围扩大
- **司马懿 - 鬼谋：** 计谋成功率+40%，计谋效果+50%，可识破敌方计谋

**强力技能：**
- **赵云 - 龙胆：** 防御时攻击力不降低，突袭成功率+50%
- **张辽 - 威震：** 战斗开始时降低敌方士气20点
- **甘宁 - 奇袭：** 突袭伤害+80%，移动速度+40%
- **黄忠 - 神射：** 弓兵攻击+60%，射程+1
- **典韦 - 恶来：** 防御力+50%，保护主公时防御+100%

### 2.6 外交系统
- **外交行动：** 联盟、宣战、和谈、停战
- **离间计：** 降低敌方武将忠诚度
- **人质系统：** 扣留敌方武将家属
- **外交关系值：** 友好、中立、敌对
- **劝降系统：**
  - **劝降武将：** 敌方武将忠诚度低于50时成功率较高
  - **劝降势力：** 敌方只剩1-2座城市时可劝降整个势力

### 2.7 间谍系统
- 派遣间谍潜入敌方城市
- 获取敌方情报：兵力、资源、武将
- 破坏活动：降低敌方城市防御或民心
- 反间谍：抓捕敌方间谍

### 2.8 历史事件系统
- 历史事件触发：如三顾茅庐、赤壁之战等
- 事件影响：获得武将、宝物、改变势力关系
- 可选择是否触发历史事件

### 2.9 剧本系统
- **黄巾之乱（184年）：** 黄巾军起义，群雄并起
- **群雄割据（190年）：** 董卓乱政，诸侯讨伐
- **官渡之战（200年）：** 曹操vs袁绍，北方争霸
- **三足鼎立（220年）：** 魏蜀吴三国鼎立

每个剧本有不同的势力分布和武将配置。

### 2.10 难度系统
- **初级：** AI智能较低，初始资源多，灾害少
- **中级：** AI智能适中，初始资源适中
- **高级：** AI智能高，初始资源少，灾害频发

### 2.11 存档系统
- 多个存档位（至少10个）
- 自动存档功能
- 存档导出/导入（JSON文件）
- 存档信息：时间、剧本、势力、回合数

### 2.12 游戏设置
- 音效和音乐开关
- 游戏速度调节
- 自动存档间隔
- UI缩放设置

### 3.2 状态管理架构
使用 Zustand 管理游戏状态，主要状态模块：

- **gameStore：** 游戏全局状态（回合、季节、剧本）
- **mapStore：** 地图状态（视野、选中城市）
- **playerStore：** 玩家数据（势力、资源、武将）
- **battleStore：** 战斗状态（战斗数据、战况）

### 3.3 数据存储架构

#### 静态数据（打包在代码中）
- 武将数据：约300-400名武将的属性、技能、初始归属
- 城市数据：城市位置、初始归属、基础属性
- 宝物数据：宝物属性、效果、初始位置
- 剧本数据：各剧本的初始配置
- 历史事件数据：事件触发条件和效果

#### 动态数据（IndexedDB存储）
- 存档数据：游戏进度、玩家决策
- 设置数据：音效、速度等用户偏好
- 缓存数据：临时计算结果

### 3.4 Canvas渲染架构
- **地图层：** 绘制地形、城市、势力范围
- **单位层：** 绘制军队、武将图标
- **效果层：** 绘制战斗特效、天气效果
- **UI层：** 绘制选中框、提示信息
- 使用 requestAnimationFrame 实现游戏循环

### 3.5 AI决策架构
- **分阶段策略：**
  - 初期（1-20回合）：优先发展经济、招募武将
  - 中期（21-50回合）：扩张领土、建立联盟
  - 后期（51+回合）：统一战争、最终决战
- **权重评分系统：** 为每个可能的行动打分
- **决策树：** 根据当前局势选择最优策略

## 五、关键算法与游戏机制

### 5.1 战斗伤害计算
```typescript
// 基础伤害公式
damage = (攻击方武力 × 兵种系数 × 地形加成 × 天气加成) 
       - (防御方统率 × 兵种克制 × 城防加成)

// 兵种克制
infantry > cavalry > archer > infantry
navy在水战中有优势

// 地形加成
mountain: 防御+30%
water: 水军+50%
pass: 防御+50%

// 武将技能影响
勇将: 伤害+20%
神算: 计谋成功率+30%
```

### 5.2 内政收益计算
```typescript
// 每回合收益
money = 人口 × (商业度/100) × (市场等级 × 0.1) × 季节系数
food = 人口 × (开发度/100) × (农场等级 × 0.1) × 季节系数
soldiers = 人口 × 0.01 × (兵营等级 × 0.1)

// 季节系数
spring: 1.0, summer: 1.2, autumn: 1.5, winter: 0.8
```

### 5.3 忠诚度变化机制
```typescript
// 忠诚度影响因素
- 发放俸禄: +5~10
- 封官: +10~20
- 赐予宝物: +10~30
- 战斗失败: -5~10
- 长期未出战: -2/回合
- 被离间: -10~20
- 主公死亡: -10~30（根据继承者魅力）
```

### 5.4 AI决策权重系统
```typescript
// AI行动评分示例
function evaluateAction(action, aiState) {
  let score = 0;
  
  // 内政行动
  if (action.type === 'develop') {
    score += aiState.money < 1000 ? 50 : 20;
    score += aiState.food < 5000 ? 40 : 15;
  }
  
  // 军事行动
  if (action.type === 'attack') {
    score += militaryStrengthRatio > 1.5 ? 60 : 20;
    score += targetCityDefense < 50 ? 30 : 10;
  }
  
  return score;
}
```

### 7.1 Canvas渲染优化
```typescript
// 分层渲染，减少重绘
class MapRenderer {
  private terrainLayer: Canvas;   // 地形层（不常变）
  private cityLayer: Canvas;      // 城市层（偶尔变）
  private unitLayer: Canvas;      // 单位层（频繁变）
  
  render() {
    // 只重绘变化的层
    if (this.terrainChanged) {
      this.renderTerrain();
    }
    if (this.cityChanged) {
      this.renderCities();
    }
    this.renderUnits(); // 单位每帧都可能变
  }
}

// 视口裁剪，只渲染可见区域
function renderVisibleArea(viewport) {
  const visibleCities = cities.filter(city => 
    isInViewport(city.position, viewport)
  );
  visibleCities.forEach(city => renderCity(city));
}
```

### 7.4 IndexedDB存储封装
```typescript
class SaveManager {
  private db: IDBDatabase;
  
  async saveGame(slotId: string, gameData: GameState) {
    const tx = this.db.transaction(['saves'], 'readwrite');
    const store = tx.objectStore('saves');
    await store.put({
      id: slotId,
      data: gameData,
      timestamp: Date.now()
    });
  }
  
  async loadGame(slotId: string): Promise<GameState> {
    const tx = this.db.transaction(['saves'], 'readonly');
    const store = tx.objectStore('saves');
    const result = await store.get(slotId);
    return result?.data;
  }
  
  async exportSave(slotId: string): Promise<string> {
    const data = await this.loadGame(slotId);
    return JSON.stringify(data);
  }
}
```
